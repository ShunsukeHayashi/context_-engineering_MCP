# Command Stack Execution Framework - 使用例\n\n## 概要\nコマンドスタック実行フレームワークは、複雑なタスクを構造化されたコマンドスタックとして実行し、段階的に成果物を生成するフレームワークです。\n\n## 基本的な使用例\n\n### プロジェクト企画書作成\n\n#### テンプレート変数設定\n```json\n{\n  \"goal\": \"新規SaaSプロダクトの企画書を作成する\",\n  \"deliverable\": \"包括的なプロダクト企画書\",\n  \"reference_example\": \"既存の成功したSaaSプロダクトの企画事例を参考に構造化された企画書\",\n  \n  \"l1_heading_1\": \"市場分析とターゲット設定\",\n  \"l1_heading_2\": \"プロダクト機能定義\", \n  \"l1_heading_3\": \"事業計画と収益モデル\",\n  \n  \"l1_1_subheading_1\": \"市場規模調査\",\n  \"l1_1_subheading_2\": \"競合分析\",\n  \"l1_2_subheading_1\": \"コア機能設計\",\n  \"l1_2_subheading_2\": \"UI/UX要件\",\n  \n  \"l1_1_1_detail_1\": \"TAM・SAM・SOM分析\",\n  \"l1_1_1_detail_2\": \"成長トレンド分析\"\n}\n```\n\n#### 実行プロセス\n\n**[C1] 構造化分析フェーズ**\n```\n上位構造 (L1):\n- 市場分析とターゲット設定\n- プロダクト機能定義\n- 事業計画と収益モデル\n\n中位構造 (L2):\n市場分析とターゲット設定:\n  - 市場規模調査\n  - 競合分析\n\nプロダクト機能定義:\n  - コア機能設計\n  - UI/UX要件\n\n下位構造 (L3):\n市場規模調査:\n  - TAM・SAM・SOM分析\n  - 成長トレンド分析\n```\n\n**[C2] インストラクション生成フェーズ**\n```\nTask 1: 市場分析とターゲット設定\nUser Prompt: SaaSプロダクト市場の規模と成長性を分析し、ターゲット顧客セグメントを明確に定義してください。TAM・SAM・SOMの数値算出も含めてください。\n期待される成果: 市場規模データ、競合マップ、ターゲットペルソナ定義\n\nTask 2: プロダクト機能定義\nUser Prompt: ターゲット顧客のペインポイントを解決するコア機能を設計し、UI/UX要件と技術仕様を詳細に定義してください。\n期待される成果: 機能仕様書、ワイヤーフレーム、技術要件\n\nTask 3: 事業計画と収益モデル\nUser Prompt: プロダクトの収益モデル、価格戦略、5年間の事業計画を策定してください。財務予測も含めてください。\n期待される成果: 収益モデル、価格設定、財務計画\n```\n\n**[C3] 連続実行フェーズ**\n```\nCommand Run 1: 市場分析とターゲット設定\n---\n[実行結果]\n- TAM: 500億円、SAM: 50億円、SOM: 5億円\n- 主要競合: Salesforce, HubSpot, Marketo\n- ターゲット: 中小企業のマーケティング担当者\n- ペルソナ: 30-40代、デジタルマーケティング経験2-5年\n\nCommand Run 2: プロダクト機能定義\n---\n[実行結果]\n- コア機能: リード管理、メール自動化、分析ダッシュボード\n- UI/UX: シンプルで直感的、モバイル対応\n- 技術仕様: React + Node.js、AWS基盤\n\nCommand Run 3: 事業計画と収益モデル\n---\n[実行結果]\n- 収益モデル: SaaS月額課金制\n- 価格設定: ベーシック9,800円/月、プロ19,800円/月\n- 5年売上予測: 1年目1,000万円→5年目10億円\n```\n\n### 技術設計書作成の例\n\n#### テンプレート変数設定\n```json\n{\n  \"goal\": \"マイクロサービスアーキテクチャの技術設計書を作成する\",\n  \"deliverable\": \"詳細な技術設計書とアーキテクチャ図\",\n  \"reference_example\": \"Netflix、Uberなどの大規模マイクロサービス実装事例\",\n  \n  \"l1_heading_1\": \"システム全体アーキテクチャ\",\n  \"l1_heading_2\": \"個別サービス設計\",\n  \"l1_heading_3\": \"インフラストラクチャ設計\",\n  \n  \"task_1_prompt\": \"マイクロサービス間の通信方式、データ整合性、障害対応を含む全体アーキテクチャを設計してください\",\n  \"task_2_prompt\": \"各マイクロサービスのAPI仕様、データモデル、処理フローを詳細に設計してください\",\n  \"task_3_prompt\": \"Kubernetes環境での運用、監視、ログ収集、CI/CDパイプラインを設計してください\"\n}\n```\n\n## 高度な使用パターン\n\n### 1. 教育コンテンツ作成\n\n```json\n{\n  \"goal\": \"プログラミング初心者向けのPython学習コースを作成する\",\n  \"deliverable\": \"12週間の構造化学習プログラム\",\n  \"l1_heading_1\": \"基礎文法習得\",\n  \"l1_heading_2\": \"実践プロジェクト\",\n  \"l1_heading_3\": \"応用技術学習\"\n}\n```\n\n### 2. マーケティング戦略立案\n\n```json\n{\n  \"goal\": \"B2B SaaSプロダクトのマーケティング戦略を策定する\",\n  \"deliverable\": \"6ヶ月のマーケティング実行計画\",\n  \"l1_heading_1\": \"ターゲット顧客分析\",\n  \"l1_heading_2\": \"チャネル戦略\",\n  \"l1_heading_3\": \"コンテンツ戦略\"\n}\n```\n\n### 3. 組織変革計画\n\n```json\n{\n  \"goal\": \"デジタル変革を推進する組織変革計画を作成する\",\n  \"deliverable\": \"12ヶ月の変革実行ロードマップ\",\n  \"l1_heading_1\": \"現状分析と課題特定\",\n  \"l1_heading_2\": \"変革戦略設計\",\n  \"l1_heading_3\": \"実行計画と成果測定\"\n}\n```\n\n## 成功のポイント\n\n### 1. 明確な構造化\n- L1（上位）: 3-5個の主要領域\n- L2（中位）: 各L1に2-3個のサブ要素\n- L3（下位）: 具体的なタスクや詳細項目\n\n### 2. 適切なタスク設計\n- 各タスクは独立して実行可能\n- 前のタスクの結果が次のタスクの入力となる\n- 成果物が明確に定義されている\n\n### 3. 品質管理\n- 各段階での成果物の品質チェック\n- 全体の整合性確認\n- ステークホルダーのレビュー\n\n## トラブルシューティング\n\n### よくある問題\n\n**問題**: タスク間の依存関係が複雑\n**解決策**: より単純な構造に分解し、依存関係を明確化\n\n**問題**: 成果物の品質が一定しない\n**解決策**: 品質基準を事前に定義し、チェックリストを活用\n\n**問題**: 実行時間が予想より長い\n**解決策**: タスクの粒度を調整し、並行実行可能な部分を特定\n\n## 関連テンプレート\n\n- **Multi-Agent Workflow**: 複数エージェントでの協調実行\n- **YAML Context Agent**: 構造化された情報抽出\n- **Business Proposal**: ビジネス文書作成\n- **Technical Documentation**: 技術文書作成
